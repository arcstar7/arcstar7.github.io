AI Solutions & Strategy

Artificial intelligence is a foundational layer for business growth, not an isolated IT experiment.

Traditional invoice processing is highly manual and prone to data entry errors. By integrating Generative AI and Machine Learning models with the ERP's financial modules, the system can automatically ingest invoices from emails or portals in various unstructured formats (PDFs, images).

AI extracts key-value pairs (vendor, amounts, tax, dates) and performs automated two-way or three-way matching against existing purchase orders and receipt logs in the ERP.

Exceptions or discrepancies are intelligently routed to human staff, while clean invoices are posted directly to the general ledger. This drastically reduces the invoice processing cycle from days to hours and virtually eliminates manual data entry.

1. Extraction Engine

This is assuming the ERP is Dynamics 365. First, we need to initialize Azure AI Document Intelligence, specifically the pre-built Invoice model.

It is highly accurate for key-value extraction without needing to train a custom LLM.

import os
from azure.core.credentials import AzureKeyCredential
from azure.ai.formrecognizer import DocumentAnalysisClient

def extract_invoice_data(invoice_file_path):
    endpoint = os.environ["AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT"]
    key = os.environ["AZURE_DOCUMENT_INTELLIGENCE_KEY"]

    document_analysis_client = DocumentAnalysisClient(
        endpoint=endpoint, credential=AzureKeyCredential(key)
    )

    with open(invoice_file_path, "rb") as f:
        poller = document_analysis_client.begin_analyze_document(
            "prebuilt-invoice", document=f
        )
    invoices = poller.result()
    
    extracted_data = {}
    for idx, invoice in enumerate(invoices.documents):
        fields = invoice.fields
        extracted_data = {
            "vendor_name": fields.get("VendorName").value if fields.get("VendorName") else None,
            "invoice_date": fields.get("InvoiceDate").value if fields.get("InvoiceDate") else None,
            "invoice_total": fields.get("InvoiceTotal").value.amount if fields.get("InvoiceTotal") else 0.0,
            "total_tax": fields.get("TotalTax").value.amount if fields.get("TotalTax") else 0.0,
            "purchase_order": fields.get("PurchaseOrder").value if fields.get("PurchaseOrder") else None,
            "line_items": []
        }
        
        # Extracting line items for 3-way matching
        if fields.get("Items"):
            for item in fields.get("Items").value:
                item_fields = item.value
                extracted_data["line_items"].append({
                    "description": item_fields.get("Description").value if item_fields.get("Description") else "",
                    "quantity": item_fields.get("Quantity").value if item_fields.get("Quantity") else 0.0,
                    "unit_price": item_fields.get("UnitPrice").value.amount if item_fields.get("UnitPrice") else 0.0
                })
                
    return extracted_data

2. Querying D365

To perform the match, you need to pull the existing Purchase Order (PO) and Product Receipt data from D365 Finance & Operations (F&O). You will use Microsoft Authentication Library (MSAL) and the D365 OData endpoints.

import requests
import msal

def get_d365_token(client_id, client_secret, tenant_id):
    authority = f"https://login.microsoftonline.com/{tenant_id}"
    app = msal.ConfidentialClientApplication(
        client_id, authority=authority, client_credential=client_secret
    )
    
    # The scope is your D365 F&O URL + /.default
    result = app.acquire_token_for_client(scopes=["https://your-d365-env.operations.dynamics.com/.default"])
    return result['access_token']

def get_d365_po_data(po_number, token):
    d365_url = f"https://your-d365-env.operations.dynamics.com/data/PurchaseOrderLines?$filter=PurchaseOrderNumber eq '{po_number}'"
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    }
    
    response = requests.get(d365_url, headers=headers)
    return response.json().get('value', [])

3. Matching Logic

Two-way matching checks if the Invoice matches the PO. Three-way matching checks if the Invoice matches the PO and the physically received goods (Receipt Log). You must implement a tolerance threshold (e.g., a 1% variance or a flat $5.00 limit) because tax rounding and shipping fees frequently cause exact matches to fail.

def perform_matching(invoice_data, po_lines, receipt_lines=None, match_type="3-way", tolerance=0.01):
    match_results = {
        "status": "Pending",
        "discrepancies": []
    }
    
    if not po_lines:
        match_results["status"] = "Failed"
        match_results["discrepancies"].append("Purchase Order not found in D365.")
        return match_results

    # Aggregate PO totals
    po_total = sum([line['LineAmount'] for line in po_lines])
    
    # 2-WAY MATCHING (Invoice vs PO)
    invoice_total = invoice_data['invoice_total']
    if abs(invoice_total - po_total) / max(po_total, 1) > tolerance:
        match_results["discrepancies"].append(f"Price mismatch: Invoice Total ({invoice_total}) vs PO Total ({po_total})")
        
    # 3-WAY MATCHING (Invoice vs PO vs Receipt)
    if match_type == "3-way":
        if not receipt_lines:
             match_results["discrepancies"].append("No product receipts found for this PO.")
        else:
            # Simplified logic: compare total quantities
            invoice_qty = sum([item['quantity'] for item in invoice_data['line_items']])
            receipt_qty = sum([line['ReceivedQuantity'] for line in receipt_lines])
            
            if invoice_qty > receipt_qty:
                match_results["discrepancies"].append(f"Quantity mismatch: Invoiced ({invoice_qty}) but Received ({receipt_qty})")

    # Determine final status
    if len(match_results["discrepancies"]) == 0:
        match_results["status"] = "Approved"
    else:
        match_results["status"] = "Exceptions Found"
        
    return match_results

4. Wrapping It Up

In a production environment, this Python logic would sit in a middleware layer (like an Azure Function or an AWS Lambda). The pipeline looks like this:

1. Invoice arrives in an email inbox.
2. An orchestrator grabs the PDF and sends it to the AI Extraction Code.
3. The extracted PO number is passed to the D365 API to pull the PO and Receipt data.
4. The Matching Code compares the two.
5. If Approved, an API call creates an AP Invoice journal in D365. If Exceptions Found, it alerts a human accountant.
